# 人工收款流程说明

v0.8.0 使用人工收款订单流程，不接 Stripe、支付宝官方接口或微信支付官方接口。

## 流程

1. 用户在商品详情页创建人工付款订单。
2. 系统生成订单号，并跳转到 `payment.html?id=订单号`。
3. 用户付款时需要在备注中填写订单号。
4. 后台管理员人工确认收款后，在后台订单管理中点击“标记已支付”。
5. `paid` 订单后续再由管理员人工执行预留卡密和演示发货流程。

## 安全边界

- 不要把真实微信或支付宝收款二维码提交到 Git。
- 不要把支付密钥、Token、API Key 或真实卡密写入项目。
- 当前不会自动发卡，也不会在页面显示真实卡密。
- 后续如需升级，可接入支付宝或微信支付官方接口，并单独设计正式支付回调和发货流程。
